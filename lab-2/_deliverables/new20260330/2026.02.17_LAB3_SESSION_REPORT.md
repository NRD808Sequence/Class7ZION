## Security Hardening Applied

Three fixes implemented and verified before teardown.

### Fix A — CloudFront Response Headers Policy (`chewbacca_rsp_security01`)

Attached to `default_cache_behavior`, covers all API responses through both origins.

| Header | Value |
|--------|-------|
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` |
| `Content-Security-Policy` | `default-src 'self'; style-src 'self' 'unsafe-inline'` |
| `X-Frame-Options` | `DENY` |
| `X-Content-Type-Options` | `nosniff` |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` |
| `Server` | `Chewbacca` (overrides origin) |

### Fix B — Flask Server Header Suppression (both user_data files)

Added `@app.after_request` handler to override `Server: Werkzeug/3.1.5 Python/3.10.12` with `Server: Chewbacca`.

### Fix C — Cache-Control on API Responses (both user_data files)

Added `Cache-Control: no-store` to all Flask responses, preventing intermediary caching of APPI-sensitive medical data.

### Post-Hardening Verification (confirmed via `curl -sI`)

```
cache-control: no-store
server: Chewbacca
x-frame-options: DENY
referrer-policy: strict-origin-when-cross-origin
content-security-policy: default-src 'self'; style-src 'self' 'unsafe-inline'
x-content-type-options: nosniff
strict-transport-security: max-age=31536000; includeSubDomains
permissions-policy: camera=(), microphone=(), geolocation=()
```
